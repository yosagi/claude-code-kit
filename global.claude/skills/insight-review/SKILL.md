# あなたのタスク: insight レポートの確認と提案の処理

ユーザーと一緒に insight レポートの提案を確認し、承認されたアクションを実行します。

## 手順

### 1. レポートの特定

inbox INDEX.md から `_from_insight.md` を探す。
見つからない場合は「未確認の insight レポートはありません」と報告して終了。

`/inbox read [ファイル名]` で内容を確認し、記載されたレポートパス（`reports/insight/YYYY-MM-DD_insight.md`）を読む。

### 2. 概要の提示

レポートの要点を簡潔に伝え、提案一覧をアクションとして整理する。

提示の形式:

```
insight レポート（YYYY-MM-DD）の提案:

1. [アクション内容]（種別: done移動）
2. [アクション内容]（種別: KB記録）
3. [アクション内容]（種別: inbox送信）
...

どれから処理しますか？番号で指定、「全部」で一括、「スキップ」で飛ばせます。
```

### 3. アクションの遂行

ユーザーの指示に従い、承認されたアクションを実行する。

**実行しうるアクション例**:

- **done/ 移動**: TODO/IDEAS ファイルを done/ に移動し、INDEX.md を更新
- **rejected/ 移動**: TODO/IDEAS ファイルを rejected/ に移動し、INDEX.md を更新
- **KB 記録**: `reports/kb/` に新規ファイルを作成し、INDEX.md に追加
- **新規 TODO/IDEAS**: ファイル作成 + INDEX.md 追加
- **inbox 送信**: /inbox-send スキルを呼び出し
- **ルール反映**: CLAUDE.md / CLAUDE.local.md への追記を提案（ユーザー確認の上で編集）
- **統合**: 複数の TODO/IDEAS を1つに統合

各アクションはユーザーの承認を得てから実行する。
アクションの実行結果を都度報告する。

### 4. 完了処理

全てのアクションが処理された（実行またはスキップ）後:

1. ユーザーに「insight レポートの処理を完了してよいですか？」と確認
2. 承認されたら `/inbox done [ファイル名]` で inbox ファイルを完了にする
3. 「insight レポートの処理を完了しました。」と報告

## 注意事項

- アクションは必ずユーザーの承認を得てから実行する
- ユーザーが「後で」「今はやらない」と言った場合はスキルを終了する（inbox は残す）
- 部分的に処理して中断した場合も inbox は残す（次回セッションで再度表示される）
- 提案の修正や追加もユーザーの指示に応じて柔軟に対応する
